# Low Precision Fortran (LPF)

Low Precision Float types in Fortran.

## Usage

Since Fortran does not provide and interface to low precision float types, like
fp16 or bfloat16. This library implements both on top of an int16 datatype, such
that they become interoperable between C and Fortran via a reinterpretation of
the pointers. Furthermore, it uses Fortran 2008 features to provide an easy to
use interface on the Fortran side. By defining a type
```fortran
 use fp16_support
 type(FP16) :: x
```
and using it like any other `REAL` variable, `fp16` can be used in Fortran. It
works like any other numerical type, but with one difference: If they are
involved in an computation with `INTERGER` or `REAL(real32)` datatypes, the
whole computation is performed in low precision.

Since Fortran does not support userdefined type literals, the low precision type
are generated by assignment or
```fortran
x = 1.0E0
x = FP16(1.0D0)
```

### IO

The datatypes only support the formated output with `WRITE` and `PRINT`
statements. If used in `FORMAT` statements, the `DT(W,P)` identifier can be
used, where `W` indicates the width of the overall number and `P` the precision.

### Math-Function

The low precision datatypes support all standard Fortran Math functions. By now,
they use their `REAL(real32)` counterpart with typecasts and the implementation
from libm intead of the ones from the Fortran compilers. In case of GNU Fortran,
this would be from mpfr.

## Installation

The library currently only works with gcc compilers supporting the `_Float16`
datatype and the `-mf16c` switch. This starts with GCC 11. For proper hardware
support, a CPU with `AVX512FP16` is required.

### Building
```shell
cmake -S . -B build-dir -DCMAKE_INSTALL_PREFIX=INSTALL-LOCATION
cmake --build build-dir
cmake --build build-dir --target install
```

### Testing
```shell
(cd build-dir; ctest --output-on-failure)
```

### Usage in other projects

The project can either be used as subproject or integrated using CMake's
`FetchContent` mechanism.
